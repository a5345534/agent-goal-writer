OpenSpec decision-review explainer

add-structured-writer-workflow-phases

此 change 將 writer workflow 從單一 value gate 擴展為 phase-aware、可恢復、可檢查的 workspace 能力，借鑑 Omnigent 的 phase planner、extract/reflect/recover、reasoning graph 與 loop guard 概念，但不引入 Omnigent 依賴。

WhyWhatBefore/AfterFlow決策實作切片驗證風險Prompts

為什麼要做

現有 value-gated workflow 已能阻止明顯不完整的目標直接寫 OpenSpec，但單一 gate artifact 不足以表示長對話中的 phase progress、重複問題、價值反思、恢復建議與 load-bearing claim preservation。

此 change 的核心是：保留 workspace-local 狀態來支撐對話與檢查，同時讓 OpenSpec markdown/spec 繼續作為唯一 governed source of truth。

決策：新增 phase-aware workflow-state、extract/reflect/recover pipeline、claim graph preservation、loop guards、registry-style organization。Omnigent 僅作為設計靈感，不新增依賴、不搬完整 ReAct framework。

What Changes

Phase-aware state

新增 workflow-state.json，追蹤 intake、value challenge、clarification、spec kernel、pre-spec gate、OpenSpec write、validation。

Extract / Reflect / Recover

把 user/source input 轉成 claims，反思 value/scope/risk/verification gaps，並輸出一個下一步建議。

Claim graph

追蹤 load-bearing claims 是否落到 proposal/design/tasks/spec，未保存則成為 gate blocker。

Before / After

變更前 Before

  • 主要依賴 value-gate.json 判斷 pre-spec readiness。
  • phase progress、recovery 建議與重複問題沒有完整結構。
  • load-bearing preservation 主要靠人工與 design notes。

變更後 After

  • .writer-workflow/changes/<change-name>/workflow-state.json 可 resume phase progress。
  • extract/reflect/recover 讓價值挑戰與下一步更可檢查。
  • claim-graph.json 可阻擋未保存的 load-bearing claims。

Scope boundaries

不包含

Decision Points 決策點

D1. 新增 workflow-state.json

Recommendation: 使用 phase/step/status/loopGuard 結構，保留 value-gate 相容。

Trade-off: 狀態較多，但能 resume 與測試。

Confidence: 高。

D2. 新增 claim graph

Recommendation: 對 load-bearing claims 建 preservation map，未保存則 gate blocker。

Trade-off: 需要維護 claims，但能降低規格遺漏。

Confidence: 中高。

D3. 不導入 Omnigent dependency

Recommendation: 借鑑模式，不複製 runtime 或依賴。

Trade-off: 需要自行實作小型版本，但保持 skill self-contained。

Confidence: 高。

Implementation slices 實作切片

Slice 1：State schema

新增 workflow-state.json 與 phase transition helpers。Acceptance：init 建立所有新 artifacts。

Slice 2：Pipeline

新增 extracted-claims、reflection-report、recovery-actions。Acceptance：recovery 恆只輸出一個 next action。

Slice 3：Claim graph + guards

新增 claim preservation blocker 與 loop limits。Acceptance：未保存 load-bearing claim 會阻擋 gate。

Risk register 風險清單

helper 過度複雜

緩解：只借鑑小型資料結構，不導入 ReAct loop。

deterministic extraction 遺漏語意

緩解：標示為 advisory，可由 agent 修正與補 assumption。

workflow state 被誤認為權威來源

緩解：文件與 spec 明確規定 OpenSpec markdown/spec 才是 source of truth。

loop guard 太早中止

緩解：保留 configurable defaults 與 fallback path。

Rollout / rollback

採 additive rollout：保留 value-gate.json，新增 workflow-state.json 與 pipeline artifacts。若新狀態造成摩擦，可暫時只使用 value-gate compatibility path，但 claim preservation 與 loop guard 應保留為下一輪目標。

Current status
