OpenSpec Change Explainer

harden-problem-scope-response-routing

決策審查頁：把 Problem & Scope Confirmation Flow 的黑箱測試結論寫成可審查的 Stage 1 流程強化變更。

為什麼需要這個變更

黑箱測試顯示，雖然文件與 helper 已定義 Stage 1.5–1.7 的 Problem & Scope Confirmation Flow，但 agent 的自然語言第一回覆仍可能在 gate 執行前提前進入 Value Challenge、推薦 no-build / smaller-scope、接受錯誤 gate decision，或聲稱 artifact 已記錄。

此變更的核心目標是讓第一回覆也遵守「scope closure before value judgment」，把提示詞規範、狀態路由、固定模板、response lint 與回歸測試串成可審查的合約。

What Changes

First-response router

在輸出分析前，先依 scope uncertainty、scope selection、Stage 1.7 decision、digest freshness 決定允許輸出的內容。

Framing-only template

Stage 1.5 僅能列中立 scope candidates 與 bounded clarification，不得推薦 no-build 或 smaller-scope。

Regression + response lint

用十個黑箱案例鎖住 premature value judgment、錯 gate decision、missing inputDigests 等行為。

Flow / 架構視覺

Affected Surfaces

  • SKILL.md：放入前置 router 與模板，且要早於一般 value challenge 指引。
  • docs/problem-scope-confirmation-flow.md：補 routing matrix 與中立候選 scope 定義。
  • prompts/ideation/*：強化 framing-only 與 bounded clarification prompt。
  • scripts/goal-spec-workflow / tests：加入 next-step、response lint 或黑箱 fixture。

Scope Boundaries

本 change 只處理 Stage 1.5–1.7 第一回覆與相關驗證。它不移除後續 value assessment；no-build / smaller-scope 仍可在 scope confirmed 且進入正確分析階段後使用。

Decision Points / 決策點

D1：Router 先於自由回覆

Recommendation: 採用 routing matrix。Trade-off 是 prompt 較硬，但能直接阻止測試失敗模式。

D2：候選 scope 不是推薦

允許列 options，但禁止 best/recommended/no-build/smaller-scope recommendation。

D3：Persisted-state honesty

沒有 artifact 就不得說 gate passed 或 recorded，只能說 would route / would record。

D4：Response lint

用 forbidden/required markers 測第一回覆，補足 helper artifact gate 之前的空窗。

Implementation Slices / 實作切片

  1. Skill/docs contract: 更新 SKILL、docs、prompts；acceptance criteria 是測試案例 1/3/4/5/9 不再違規。
  2. Helper routing/lint: 新增或強化 next-step / response lint；acceptance criteria 是 Stage 1.7 invalid approval 被拒絕。
  3. Regression tests: 加十個黑箱 fixture；dependencies 是先確認 router semantics。

Verification Plan / 驗證計畫

  • Run scripts/test-workflow。
  • Run black-box Problem & Scope Confirmation Flow tests against installed skill path。
  • Validate source-manifest.json。
  • Confirm missing inputDigests fails closed。

Risk Register / 風險清單

High-risk：lint 過度阻擋

Mitigation：允許 forbidden terms 出現在「Not doing yet」或 invalid-decision 說明中，只阻擋 recommendation / value judgment。

Medium：agent 忽略模板

Mitigation：將模板放在 skill 前段，並用 installed-skill 黑箱測試驗證。

Rollout

先更新 OpenSpec change 與來源 manifest；實作時依序更新 skill/docs、helper/lint、tests。既有 artifact schema 盡量保持相容，新增欄位需 optional 或 schema-backed。
